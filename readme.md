1. OVERVIEW
Traditional image restoration models (like standard U-Nets) suffer from 
"Task Interference" or conflicting gradients when forced to handle multiple 
degradations simultaneously (e.g., smoothing rain vs. sharpening blur). 
This project implements an architecture inspired by AirNet (CVPR 2022) to 
dynamically route image features and solve this bottleneck.

2. ARCHITECTURE (AirNet-Lite)
- The Scout: A pre-trained ResNet18 analyzes the texture of the degraded 
  image and generates a 64-dimensional Degradation Vector.
- The Switchboard: The U-Net bottleneck multiplies the image features by 
  this scout vector, dynamically routing the math to the correct pathways.
- The Decoder: Uses Bilinear Upsampling instead of Transposed Convolutions 
  to completely eradicate checkerboard artifacts.

3. FILE STRUCTURE
HERITAGE_RESTORATION/
|-- data/
|   |-- DIV2K_train_HR/          (Your 800 pristine clean images)
|   |-- train_mixed_degraded/    (Generated by script)
|   |-- val_clean/               (50 unseen validation images)
|   `-- val_degraded/            (50 unseen validation degraded images)
|-- models/
|   `-- airnet_lite.py           (The neural network architecture)
|-- generate_data.py             (Creates synthetic rain/blur/haze data)
|-- train.py                     (Self-monitoring trainer with Early Stopping)
|-- inference.py                 (Generates side-by-side restoration image)
|-- evaluate.py                  (Calculates PSNR/SSIM on validation set)
|-- visualize.py                 (Generates a grid graphic for presentations)
`-- requirements.txt             (CUDA-enabled environment dependencies)

4. EXECUTION ORDER
Step 1: Install Dependencies
> pip install -r requirements.txt

Step 2: Generate the Chaotic Dataset
> python generate_data.py

Step 3: Train the Architecture
> python train.py
Note: Features Early Stopping. It will safely halt and save "airnet_lite_best.pth" 
before overfitting occurs.

Step 4: Objective Evaluation
> python evaluate.py
Calculates the exact PSNR and SSIM scores on your hidden validation data.

Step 5: Visual Presentation
> python visualize.py
> python inference.py
Generates presentation-ready graphics to prove the architecture's success.
